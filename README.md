# Симуляция цепного удаления элементов («Шарики»)

Реализация отказоустойчивого консольного приложения на языке C++ для моделирования игровой механики каскадного удаления одноцветных элементов в линейных структурах данных.

##  Постановка задачи

В линию выставляется последовательность шариков различных цветов, представленных в виде цифровых идентификаторов от `0` до `9`. При возникновении непрерывной цепочки, состоящей из **трех и более** шариков одного цвета, эта группа элементов аннигилирует (уничтожается), а оставшиеся части последовательности сдвигаются. 

Сдвиг элементов может спровоцировать повторное выполнение условия аннигиляции, формируя лавинообразный рекурсивный процесс схлопывания цепочек. 

* **Входные данные:** Общее количество элементов $n$ ($1 \le n \le 10^5$), после чего подается сама последовательность.
* **Выходные данные:** Количество шариков, которое будет уничтожено в результате всех последовательных каскадных схлопываний.

##  Оптимизация и сложность алгоритма

Наивный алгоритм с многократным сканированием массива и физическим удалением элементов привел бы к временной сложности $O(n^2)$, что неприемлемо при $n = 10^5$ (вызовет *Time Limit Exceeded*).

В данном проекте задача решена за **линейное время $O(n)$** и **память $O(n)$** с использованием структуры данных **динамический стек** (`std::stack`), хранящий пары вида `std::pair<int, int>`:
* `first` — идентификатор цвета шарика.
* `second` — текущее количество подряд идущих элементов данного цвета.

Применение механизма **LIFO** (Last In, First Out) позволяет агрегировать одноцветные группы «на лету» и мгновенно выполнять схлопывание за $O(1)$, автоматически соединяя края оставшейся цепочки без сдвига элементов в памяти.

##  Структура проекта и архитектура

Проект спроектирован по модульному принципу и разделен на логические уровни:

* **`main.cpp`** — Точка входа в приложение. Отвечает за инициализацию окружения и установку кодировки UTF-8 для корректного отображения кириллицы в консоли Windows.
* **`menu.cpp` / `menu.h`** — Подсистема жизненного цикла приложения, координирующая работу главного меню и выбор пользовательских сценариев.
* **`validator.cpp` / `validator.h`** — Статический класс `InputValidator`, обеспечивающий отказоустойчивость программы (`Fault Tolerance`). Перехватывает сбои потока ввода (`std::cin.fail()`), очищает буфер и проверяет диапазоны значений.
* **`classes.cpp` / `classes.h`** — Ядро бизнес-логики. Инкапсулирует класс `BallRemover`, управляющий стеком пар и вычисляющий итоговое количество уничтоженных элементов.
* **`task.cpp` / `task.h`** — Реализация режимов работы (ручной ввод последовательности или случайная генерация) с цветовой визуализацией элементов через Windows WinAPI (`SetConsoleTextAttribute`).
